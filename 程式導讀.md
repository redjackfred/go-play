## 1. 專案架構總覽

```
board.py   純粹棋盤規則 (落子、提子、ko、計分)
ai.py      神經網路 + MCTS 搜尋 (AI 大腦)
train.py   自我對弈訓練 + SGF 棋譜學習
gui.py     pygame 圖形介面
main.py    命令列入口點
```

三個核心層次：

```
棋局狀態 (GoEngine)
    ↓
盤面編碼 encode_board()  →  (10, 9, 9) 張量
    ↓
神經網路 GoNet            →  策略分布 + 局面評估值
    ↓
MCTS 搜尋                →  最佳落點
```

## 3. `board.py` — 棋盤規則引擎

### 常數

```python
EMPTY = 0   # 空格
BLACK = 1   # 黑棋
WHITE = 2   # 白棋
BOARD_SIZE = 9
KOMI = 6.5  # 白棋貼目（中國規則）
```

### `GoEngine` 主要屬性

| 屬性 | 說明 |
|------|------|
| `board` | `9×9` 二維串列，值為 EMPTY/BLACK/WHITE |
| `current_player` | 目前輪到誰下 |
| `previous_board` | 上一手落子前的棋盤，用於 ko 檢查 |
| `consecutive_passes` | 連續 pass 次數；達到 2 → 終局 |
| `game_over` | 遊戲是否結束 |
| `winner` | 勝者 |

### 核心方法

```python
engine.play(row, col)      # 落子。回傳 True/False（合法/違規）
engine.is_legal(row, col)  # 合法性檢查（自殺、ko）
engine.pass_move()         # 虛手（pass）
engine.decline_pass()      # 拒絕對手的 pass（GUI 用）
engine.resign()            # 投降，對手直接勝利
engine.get_score()         # 中國規則計分，回傳 dict
engine.get_legal_moves()   # 所有合法落點串列
engine.clone()             # 快速深度拷貝（MCTS rollout 用）
```

### 關鍵實作細節

**提子**（`play`）：落子後，對每個相鄰的對手連通組做 BFS，若氣為零則移除。

**合法性**（`is_legal`）：在真實棋盤上模擬一次（就地），完成後還原。同時檢查：

- 自殺：落子後自己的組無氣
- Ko：棋盤狀態與 `previous_board` 相同

**計分**（`get_score`）：中國規則 = 己方棋子數 + 己方圍住的空格數 + 貼目（白棋）。
空格地域以 flood-fill 判斷，僅與單色接觸的空格才屬於那色。

## 4. `ai.py` — AI 核心

這是最複雜的檔案，包含三大模組：盤面編碼、神經網路、MCTS。

### 4-1. 空間先驗熱圖（金角銀邊草肚皮）

```python
_LINE_SCORES = [0.35, 0.75, 1.70, 1.40, 0.55]
#               邊緣  邊    3線  4線   中央
```

`_make_spatial_prior()` 將 `_LINE_SCORES[dr] × _LINE_SCORES[dc]` 填入 9×9 網格，
再對整體取平均值正規化（mean=1.0），使乘法不改變整體策略的尺度，
只在落點間做相對調整。

結果：角落（3-3）的倍率約為 **3×**，中央約為 **0.3×**。

### 4-2. 盤面編碼 `encode_board(engine)`

輸出：`(10, 9, 9)` float32 張量，10 個特徵平面：

| 平面 | 意義 |
|------|------|
| 0 | 己方棋子位置 |
| 1 | 對手棋子位置 |
| 2 | 當前玩家旗標（1=黑方，0=白方） |
| 3 | 己方「氣=1」的組（叫吃中，需逃脫） |
| 4 | 己方「氣=2」的組 |
| 5 | 對手「氣=1」的組（可立即提子） |
| 6 | 對手「氣=2」的組 |
| 7 | 合法落點 |
| 8 | 潛在己方真眼 |
| 9 | 潛在對手真眼 |

平面 8、9 的計算：對所有空格，若其四個正交鄰居全是己方（或對方）棋子，
則該格為潛在眼。邊緣/角落的棋盤外格視為「不算鄰居」，所以角落只需 2 個己方鄰居即成眼。

### 4-3. 神經網路 `GoNet`

```
輸入 (10, 9, 9)
  ↓  Stem: Conv3×3 → BN → ReLU
  ↓  Trunk: 5 個 ResBlock（128 通道）
  ┌─────────────────────────────────────┐
  │ 策略頭 (Policy Head)                 │
  │ Conv1×1(2ch) → BN → ReLU           │
  │ Flatten → FC → log-softmax over 81  │
  └─────────────────────────────────────┘
  ┌─────────────────────────────────────┐
  │ 評估頭 (Value Head)                  │
  │ Conv1×1(1ch) → BN → ReLU           │
  │ Flatten → FC(128) → ReLU → FC(1)    │
  │ tanh → [-1, +1]                     │
  └─────────────────────────────────────┘
```

- **策略頭**：輸出 81 個落點的對數機率；在 MCTS 中作為 PUCT 的先驗 P(s,a)。
- **評估頭**：從「目前玩家」視角輸出局面分數；+1 代表己方必勝，-1 代表必敗。

約 **160 萬** 個參數。

### 4-4. MCTS 搜尋

#### 節點 `MCTSNode`

```python
prior        # 先驗機率 P(s,a)（來自神經網路）
visit_count  # 被訪問次數 N(s,a)
value_sum    # 累計評估值（用來算平均 Q）
children     # Dict[move → MCTSNode]
```

#### PUCT 公式

```
PUCT(s, a) = -Q(s,a) + c_puct × P(s,a) × sqrt(N(s)) / (1 + N(s,a))
```

- `-Q(s,a)`：從父節點視角，子節點的 Q 值要取負號（零和博弈）。
- 探索項 U：先驗高且被訪問少的子節點有大 U，引導搜尋探索未知落點。
- `c_puct`（預設 1.5）：控制開發 vs. 探索的平衡。

#### 批次模擬 `_run_batch`

MCTS 每輪跑 `eval_batch_size`（預設 8）條路徑，最後只呼叫一次神經網路。
關鍵技術：**虛擬損失（Virtual Loss）**：

```
Phase 1: 沿樹走到葉節點，每個節點 visit_count += 1, value_sum += 1
         → Q 值暫時偏高（看起來已被探索），其他路徑轉向不同子節點
Phase 2: 批次呼叫神經網路，展開葉節點
Phase 3: 取消虛擬損失，用真實評估值反向傳播
         每往上一層，value 取負號（視角翻轉）
```

#### 先驗調整 `_create_children`

子節點建立時，對策略先驗做**乘法**調整（不覆蓋，僅放大縮小）：

| 調整項 | 條件 | 倍率 |
|--------|------|------|
| 空間先驗 | 常態 | `1 + spatial_bias × (heatmap - 1)` |
| 眼位加成 | 創造己方眼 | `× (1 + eye_bias × 眼數)` |
| 反眼加成 | 破壞對手眼 | `× (1 + anti_eye_bias × 眼數)` |
| 提子加成 | 對手氣=1（僅對弈）| `× capture_bias` (×5) |
| 逃脫加成 | 己方氣=1（僅對弈）| `× atari_escape_bias` (×4) |
| 連接懲罰 | ≥2鄰己方、0鄰對方（僅對弈）| `× connection_penalty` (×0.35) |

「僅對弈」的意思：`self.training=False` 才啟用。
訓練時保持策略分布自然，讓神經網路從遊戲結果中學習戰術。

#### Pass 邏輯

```
訓練模式：
    pass_raw = value × pass_weight × (5.0 if 已有一次pass else 1.0)
    → 兩方自由 pass，確保自我對弈遊戲能正常終局

對弈模式：
    pass_raw = value × pass_weight × 5.0  （只在對手剛 pass 後才算）
    → AI 不主動 pass；若對手 pass 且己方贏面大，接受 pass 終局
```

#### `select_move` 優先順序（對弈模式）

```
1. 提子  ← 對手有叫吃，直接提（確定性）
2. 逃吃  ← 己方被叫吃，只有一個逃點且非無效逃脫（確定性）
3. 開局角落  ← 己方棋子少於 4 顆時，依序搶 (2,2)→(6,6)→(2,6)→(6,2)
4. MCTS  ← 一般情況跑模擬取勝率最高的落點
```

訓練模式跳過步驟 1–3，直接進入 MCTS。

#### 過濾器

- `_filter_eyes`：從合法落點中移除己方真眼。若所有落點都是眼，回傳空串列，呼叫者自動 pass。
- `_filter_dead_moves`：移除「死亡區域」的落點——空格少於 4 格、只被對手包圍的封閉區域，落子必死。例外：若該格同時是提子點，仍保留。

#### 輔助函式

| 函式 | 說明 |
|------|------|
| `_iter_groups(board)` | 生成器：對每個連通組 yield `(顏色, 組格子, 氣格子)` |
| `_get_tactical_moves(board, color)` | 回傳 `(可提子集合, 可逃叫吃集合)` |
| `_is_own_eye(board, r, c, color)` | 判斷 (r,c) 是否為 color 的真眼 |
| `_is_wasteful_connection(board, r, c, color)` | 判斷是否為無謂連接（懲罰用） |
| `_escape_is_futile(engine, move)` | 單步前瞻：逃脫後仍在叫吃且對手可立即提子 → 放棄逃跑 |
| `_find_dead_zone_cells(board, color, max_size=4)` | 找出被對手包圍的死亡空格 |
| `_eye_score(board, r, c, color)` | 計算落在 (r,c) 後創造幾個眼/破壞幾個對手眼 |

## 5. `train.py` — 訓練流程

### 兩種訓練模式

#### 模式 A：SGF 棋譜學習（冷啟動）

```bash
python train.py sgf --sgf-dir ./sgf_files --epochs 20 --out model_cold.pt
```

從職業或業餘棋譜（`.sgf` 格式）提取每手的盤面狀態，以棋手的落點作為策略目標，
訓練網路模仿人類下法。這給網路一個好的初始權重，避免從亂下起步。

#### 模式 B：自我對弈（AlphaZero 風格）

```bash
python train.py selfplay --games 500 --sims 50 --workers 4 --out model_v6.pt
```

流程：

```
1. 生成 N 場 AI 自我對弈
   ↓
2. 每一步記錄 (盤面, MCTS策略分布, 終局評估)
   ↓
3. 對範例做 16× 資料增強（8 個幾何對稱 × 2 顏色互換）
   ↓
4. 用這些範例訓練網路（交叉熵 + MSE）
   ↓
5. 重複直到 games 完成；最後做最終深度訓練
```

### 評估目標值（Value Target）

```python
_SCORE_NORM = 40.0
value_target = (black_score - white_score) / 40.0   # 從黑方視角
# 若當前玩家是白方，取負號
```

用**分差**而非二元勝負（±1）的原因：只靠貼目贏的局（僅 ~6.5 分差）的目標值約為 0.16，
而非 ±1。這打破了「白方小勝 → 模型更偏好白方 → 白方更常勝」的正回饋迴路。

### 16× 資料增強

```python
# 8 個幾何變換（旋轉 × 翻轉）
_AUG_TRANSFORMS = [(flip_lr, num_rot90), ...]

# 顏色互換（黑白交換）
_COLOR_SWAP = [1, 0, None, 5, 6, 3, 4, 7, 9, 8]
# 平面 2（當前玩家旗標）取反；評估值取負號
```

### 訓練細節

| 項目 | 值 |
|------|----|
| 損失函式 | 策略：cross-entropy；評估：MSE；合計 = p_loss + v_loss |
| 最佳化器 | Adam，lr=5e-4（最終深度訓練降至 3e-4） |
| 批次大小 | 256 |
| Replay buffer | 20,000 範例（先進先出） |
| 自我對弈溫度 | 前 12 手 T=1.0（隨機），之後 T=0.0（最佳） |
| 並行 workers | 多進程產生遊戲（module-level function 才能被 pickle） |

## 8. 資料流程圖

### 推論（對弈時）

```
玩家輪到 current_player
  │
  ├─ 有對手氣=1 的組？ → 直接提子（跳過 MCTS）
  ├─ 己方氣=1，唯一逃點且非無效？ → 直接逃（跳過 MCTS）
  ├─ 己方棋子少於4顆？ → 搶 3-3 角落（跳過 MCTS）
  │
  └─ 否則：MCTS 搜尋
       │
       └─ repeat N 次（批次大小 8）：
            1. 從根節點走到葉節點（PUCT 選子，套虛擬損失）
            2. 批次呼叫 GoNet → (policy, value)
            3. 展開葉節點，調整先驗
            4. 取消虛擬損失，反向傳播 value
       │
       └─ 取訪問次數最高的子節點落點
```

### 訓練時

```
多進程 workers 並行跑自我對弈
  │
  └─ 每步：GoNet(board) → policy; MCTS; 執行最佳落點
  └─ 終局：計算 (black_score - white_score) / 40.0
  └─ 回傳 [(盤面, MCTS策略, value)] × 所有步數
  │
主進程收集進 replay buffer（20,000 範例 FIFO）
  │
16× 增強 → 隨機抽 batch=256
  │
loss = cross_entropy(pred_policy, target_policy)
      + mse(pred_value, target_value)
  │
Adam.step()  →  更新 GoNet 權重
```

## 9. 重要參數說明

### MCTS 參數（`create_ai` / `MCTS.__init__`）

| 參數 | 預設 | 說明 |
|------|------|------|
| `num_simulations` | 400 | 每步跑幾次 MCTS 模擬。越多越強但越慢。 |
| `c_puct` | 1.5 | 探索係數。越大越傾向訪問低 Q、高 prior 的節點。 |
| `spatial_bias` | 1.0 | 空間先驗強度。0=關閉，正值越大越偏好角落/3線。 |
| `capture_bias` | 5.0 | 提子落點的先驗倍率（僅對弈）。 |
| `atari_escape_bias` | 4.0 | 逃叫吃落點的先驗倍率（僅對弈）。 |
| `connection_penalty` | 0.35 | 無謂連接的先驗懲罰倍率（僅對弈）。 |
| `eye_bias` | 1.0 | 每創造一個己方眼的先驗加成。 |
| `anti_eye_bias` | 0.8 | 每破壞一個對手眼的先驗加成。 |
| `pass_weight` | 0.15 | 控制 pass 節點的基礎先驗權重。 |
| `training` | True | 訓練模式：開啟 Dirichlet 雜訊、關閉戰術偏差。 |
| `resign_threshold` | None | None=不投降；設成 -0.9 等閾值則在大劣時投降。 |

### 訓練參數（`train.py`）

| 參數 | 預設 | 說明 |
|------|------|------|
| `--games` | 500 | 自我對弈場數 |
| `--sims` | 200 | 每步 MCTS 模擬數（越少越快，但弱） |
| `--workers` | 1 | 並行遊戲生成 worker 數 |
| `--steps-per-game` | 4 | 每收到一批遊戲後的梯度更新步數 |
| `_SCORE_NORM` | 40.0 | 分差正規化因子（±40分 → ±1.0） |

## 10. 附錄：棋理對應

| 棋理 | 程式實作 |
|------|----------|
| 金角銀邊草肚皮 | `_LINE_SCORES` + `_SPATIAL_PRIOR` + `_OPENING_PRIORITIES` |
| 先提再逃（戰術優先） | `select_move` 步驟 1-2 |
| 不填真眼 | `_filter_eyes` + `_is_own_eye` |
| 不落死地 | `_filter_dead_moves` + `_find_dead_zone_cells` |
| 無謂補棋是緩手 | `_is_wasteful_connection` + `connection_penalty` |
| 無效逃脫不如棄子 | `_escape_is_futile` |
| 正確貼目 | `KOMI = 6.5` |
| 防止白方優勢正回饋 | 分差 value target (`_SCORE_NORM`) |
